## Zero-Day Vulnerability Fixes

### 13. Homogeneous Model Monoculture Bias

Problem: PI and Red Team use same model, creating "echo chamber" effect.

Fix:
- File: src/core/config_loader.py, src/core/zero_day_defense.py
- Solution:
  1. HeterogeneousModelPool for different models per agent
  2. Environment variables:
     - PI_MODEL=claude-sonnet-4-6
     - RED_TEAM_MODEL=claude-haiku-4-5-20251001
  3. Auto-verify heterogeneity

Test Result: [OK] Heterogeneous pool initialized

---

### 14. Indirect Prompt Injection via Abstracts

Problem: PubMed abstracts may contain injection attacks.

Fix:
- File: src/core/zero_day_defense.py
- Solution:
  1. PromptInjectionDefender detects injection patterns
  2. Delimiter isolation: <external_data>...</external_data>
  3. Role固化: Explicit instruction for data-only interpretation

Test Result: [OK] Injection detection works, safe wrapping (528 chars)

### 15. Lost in the Middle & Constraint Dilution

Problem: LLM "forgets" middle constraints like "no scRNA-seq".

Fix:
- File: src/core/zero_day_defense.py, src/agents/red_team_agent.py
- Solution:
  1. SteelStampReinforcer - constraint stamper
  2. Dynamic reminder at end of EACH prompt
  3. Confirmation checkpoint

Test Result: [OK] Constraints added (+638 chars)

---

### 16. Temporal API Drift

Problem: PubMed data changes in real-time, breaking reproducibility.

Fix:
- File: src/core/zero_day_defense.py, src/utils/pubmed.py
- Solution:
  1. APICacheManager - physical cache of API responses
  2. Hash-based lookup: same params = same cache
  3. Disk persistence: cache/api_responses/*.json

Test Result: [OK] Cache works, PMID count: 2
