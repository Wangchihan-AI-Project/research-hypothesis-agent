## 四、关键问题诊断与建议

### 4.1 🔴 P1: DefenseCommittee 过度严苛

**问题**: UK Biobank+XGBoost 糖尿病预测是经典的、可发表的学术课题，被判定"致命缺陷"而拒绝。

**根因**: 委员会审查协议中，"技术防范措施与数据类型不匹配"等判定标准过于抽象，缺乏量化阈值。

**建议**:
```python
# V7.4-G 优化方向：引入"科学底线"评分
SCIENTIFIC_BASELINE_SCORES = {
    'UK_BIOBANK_VALID': 0.85,   # UK Biobank + ML 类课题默认高分
    'STANDARD_ML_PIPELINE': 0.75,  # 标准机器学习流程
    'NOVEL_BUT_FEASIBLE': 0.70,    # 新颖但可行
}

# 如果 baseline_score > 0.70，委员会应倾向于 conditional_pass
```

---

### 4.2 🔴 P2: PSEUDOSCIENCE 补丁模板缺失

**问题**: 自愈引擎检测到 `PSEUDOSCIENCE` 攻击类型，但无对应补丁模板。

**根因**: `SynthesisPatchingEngine` 的 `PATCH_TEMPLATES` 字典中未定义 `PSEUDOSCIENCE` 类型。

**建议**:
```python
# V7.4-G 补丁模板扩展
PATCH_TEMPLATES = {
    'PSEUDOSCIENCE': {
        'template_id': 'PATCH_PHYSICAL_ANCHOR',
        'content': {
            'signal_capture': '补充物理传感器（测序仪/光谱仪/电极）',
            'effect_measurement': '补充可量化效应指标（生存率/响应率）',
            'experimental_path': '补充实验验证方案（样本量/时间线）',
        },
        'applies_to': ['consciousness_field', 'quantum_magic', 'energy_violation'],
    },
}
```

---

### 4.3 🟡 P3: attack_types 最终传递缺失

**问题**: 自愈引擎内部正确检测 `['BIAS', 'VALIDATION', 'LEAKAGE']`，但最终摘要显示为空 `[]`。

**根因**: `SearchSupplementAgent.execute()` 返回结果中 `attack_types` 字段未正确传递到 `BlindTestRunner` 的摘要聚合逻辑。

**建议**: 检查 `full_pipeline_tester.py` 中 `healing_result` 的字段提取逻辑。

---

### 4.4 🟡 P4: PSEUDOSCIENCE 误判攻击类型

**问题**: AlphaFold3 前沿科学被误判为 `PSEUDOSCIENCE` 攻击类型。

**根因**: `_extract_attack_types` 的关键词检测中，"物理验证"等词汇被映射到 `PSEUDOSCIENCE`。

**建议**: 优化 `_extract_attack_types` 的判定逻辑：
```python
# 区分"前沿科学需要物理验证"与"伪科学模式"
if 'physical validation' in rejection_reason.lower():
    if 'alphafold' in hypothesis_text.lower() or 'protein' in hypothesis_text.lower():
        attack_type = 'VALIDATION'  # 前沿科学 → 方法论验证
    else:
        attack_type = 'PSEUDOSCIENCE'  # 无传感器逻辑 → 伪科学
```
