## 现在开始使用

### 第一步：配置API密钥

编辑 `.env` 文件：

```bash
cd /c/Users/PC/research-hypothesis-agent
nano .env
```

填写你的Anthropic API密钥：

```env
ANTHROPIC_API_KEY=sk-ant-xxxxx你的密钥
```

### 第二步：安装依赖

```bash
pip install -r requirements.txt
```

### 第三步：启动系统

```bash
python main.py
```

## 完整工作流程示例

1. **开始新研究流程**
   - 输入关键词：`machine learning genomics`
   - 设置最大论文数：`50`

2. **浏览和选择论文**
   - 分页浏览所有50篇论文
   - 查看感兴趣的论文详情
   - 选择10-15篇高质量论文

3. **生成假设**
   - 设置研究领域：`计算生物学`
   - 关注领域：`深度学习,基因组变异检测`
   - 系统生成3-5个研究假设

4. **验证假设**
   - 选择"验证全部假设"
   - 系统批量验证并评分
   - 自动排序显示最佳假设

5. **导出报告**
   - 查看最佳假设的完整报告
   - 导出Markdown格式报告
   - 报告包含所有技术细节和来源论文

## 配置优化建议

### 模型选择

编辑 `config.yaml` 可以选择不同模型：

```yaml
agents:
  hypothesis:
    model: "claude-opus-4-6"  # 最强大的模型
  validation:
    model: "claude-sonnet-4-6"  # 平衡性能和成本
```

### 搜索数量

```yaml
agents:
  paper_search:
    max_results: 100  # 增加论文数量
```

## 常见问题

**Q: 如何选择最佳论文数量？**
A: 建议10-20篇高质量论文，太少信息不足，太多会降低生成质量。

**Q: 验证失败怎么办？**
A: 检查API密钥、网络连接，或降低模型要求（使用Sonnet代替Opus）。

**Q: 假设质量如何保证？**
A: 系统使用Claude高质量模型生成，但建议人工复核后再投入研究。

**Q: 报告可以导出其他格式吗？**
A: 目前支持Markdown，可以轻松转换为PDF、Word等格式。
